# GameMode Lock

NeoForge 1.21.1 専用。`/gamemode` コマンドと F3+F4 によるゲームモード変更を
完全に無効化するだけの軽量Mod。設定ファイルなし、導入するだけで動作します。

## 機能

- `/gamemode` を入力すると「このワールドでは無効です」と表示して拒否
  （`/gamemode survival` のように引数を付けても同様に拒否されます）
- F3+F4 によるゲームモード切り替えも無効化
- サーバー側の処理でブロックしているため、シングルプレイをLAN公開した場合でも
  ゲームモードは変更できません
- config不要（jarを`mods`フォルダに入れるだけ）

## ビルド方法

1. Java 21 (JDK) をインストール
2. プロジェクトルートで以下を実行:

```bash
# Windows
gradlew.bat build

# macOS / Linux
./gradlew build
```

初回はNeoForgeの依存関係ダウンロードに数分かかります。
（`archive.ubuntu.com`等のパッケージ取得や、Gradle Wrapperのjarファイル自体が
このサンドボックス環境にはまだ用意されていないため、実行はご自身の環境で
行ってください。`gradle wrapper`コマンドで wrapper 一式を生成できます）

3. ビルドが成功すると `build/libs/gamemodelock-1.0.0.jar` が生成されます

## インストール方法

1. NeoForge 1.21.1 をインストール済みの Minecraft に対して、
   生成された `gamemodelock-1.0.0.jar` を `mods` フォルダに入れる
2. サーバー・クライアント両方に導入してください（サーバー側のみでも
   コマンド拒否・F3+F4拒否は機能しますが、統合サーバー(シングルプレイ)を
   使う場合はクライアント側にも同梱のMod本体が必要です）

## 仕組み（技術メモ）

- `RegisterCommandsEvent` でバニラの `/gamemode` コマンドノードを
  コマンドツリーから除去し、常に失敗メッセージを返すだけの
  同名コマンドに差し替えています。
- `ServerGamemodeSwitchEvent` をキャンセルすることで、F3+F4や
  他Mod経由など、コマンド以外の経路からのゲームモード変更要求も
  サーバー側で一律ブロックしています。
- どちらもサーバー側（統合サーバーを含む）で処理されるため、
  LAN公開時の設定ダイアログで操作しても実際の変更は行われません。
